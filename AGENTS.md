# Project Operating Contract

- For non-trivial work, start with a plan before coding.
- Prefer minimal, local changes over broad rewrites.
- Respect existing patterns before introducing new abstractions.
- When behavior changes, update or add tests.
- If requirements are ambiguous, ask clarifying questions first.
- Do not expose secrets, disable auth, or weaken validation.
- If touching sensitive data, check compliance guidance in `docs/compliance.md`.
- Before finishing, run the relevant lint, typecheck, and tests.
