---
name: feature-intake
description: Converts a rough request into implementation-ready scope, acceptance criteria, risks, and test cases.
---

When a user asks for a new feature or change request:

1. Summarize the problem in one paragraph.
2. Write acceptance criteria.
3. List edge cases.
4. Identify impacted areas.
5. Separate scope from non-goals.
6. Suggest whether architect, frontend, backend, or compliance subagents should be used.
